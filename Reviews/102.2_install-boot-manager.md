# Answers Evaluation
### ✅ **LPIC-1 102.2: Install a Boot Manager — Answer Key**

|#|Your Answer|✅ Correct|Explanation|
|---|---|---|---|
|**1**|✅ **b**|✅|`grub-install /dev/sda` installs GRUB to the MBR (for BIOS) or EFI system partition (for UEFI). `/dev/sda1` would install it to a partition, not the disk MBR.|
|**2**|✅ **c**|✅|After editing `/etc/default/grub`, regenerate the config using `update-grub` (Debian/Ubuntu) or `grub-mkconfig -o /boot/grub/grub.cfg` (manual method).|
|**3**|✅ **c**|✅|The main GRUB2 configuration file for BIOS systems is `/boot/grub/grub.cfg`. The old GRUB (Legacy) used `/boot/grub/menu.lst`.|
|**4**|✅ **b**|✅|`/etc/default/grub` holds environment variables like `GRUB_TIMEOUT`, `GRUB_CMDLINE_LINUX_DEFAULT`, and others that affect how `grub-mkconfig` builds the main config.|
|**5**|✅ **b**|✅|Chainloading means one bootloader loads another (e.g., GRUB loading Windows Boot Manager). Common in dual-boot systems.|
|**6**|✅ **a**|✅|In UEFI systems, the GRUB EFI binary typically resides at `/boot/efi/EFI/grub/grubx64.efi` or `/boot/efi/EFI/<distro>/grubx64.efi`.|
|**7**|❌ **a**|➡️ **b**|`grub-reboot` sets the next boot entry _once_. `grub-set-default` permanently changes the default entry until changed again.|
|**8**|✅ **a**|✅|`/boot` must be mounted before installing GRUB; otherwise, it cannot write its boot files correctly.|
|**9**|❌ **a**|➡️ **b**|The correct file for custom entries is `/boot/grub/custom.cfg`, which GRUB includes automatically if present.|
|**10**|❌ **a**|➡️ **c**|`grub-editenv list` shows the GRUB environment variables currently stored — including boot entries and defaults in use.|
